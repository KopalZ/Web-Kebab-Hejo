Cara Frontend Terhubung ke Backend

Kedua frontend (landing page dan admin dashboard) menggunakan file api.js yang sama:

// File: frontend/src/api.js  DAN  admin-dashboard/src/api.js
export const API_BASE = import.meta.env.VITE_API_BASE
  || 'https://backend-kebab-production.up.railway.app'

Cara kerjanya:

  1. Saat development lokal, buat file .env berisi VITE_API_BASE=http://localhost:3000
  2. Saat sudah deploy, ubah VITE_API_BASE di Vercel ke URL Railway backend
  3. Jika tidak ada env variable, otomatis pakai URL Railway sebagai fallback

Bagian 3 — Deploy Backend ke Railway

Backend menggunakan Express.js + Prisma ORM + PostgreSQL. Deploy dilakukan melalui CLI Railway.

3.1 Login ke Railway

railway login

Akan membuka browser untuk autentikasi. Setelah berhasil, terminal akan menampilkan pesan sukses.

3.2 Buat Project Baru (Pertama Kali Saja)

railway init

Beri nama project, misalnya: kebab-hejo

Catatan: Jika project sudah ada, skip langkah ini. Cukup hubungkan ke project yang sudah ada dengan: 
railway link
Pilih project dan service yang sudah ada.

3.3 Buat Service Backend

cd backend-kebab
railway service create backend-kebab

3.4 Tambah Database PostgreSQL

Di dashboard Railway (https://railway.app):

  1. Buka project kamu
  2. Klik "+ New" → "Database" → "Add PostgreSQL"
  3. Tunggu database selesai dibuat (sekitar 1 menit)
  4. Klik database → tab "Connect" → copy DATABASE_URL

Atau via CLI:

railway add

Pilih PostgreSQL.

3.5 Set Environment Variables

Di dashboard Railway, buka service backend-kebab → tab "Variables":

VariabelNilaiKeterangan
DATABASE_URL postgresql://user:pass@host:port/dbname Otomatis tersedia setelah tambah PostgreSQL. Klik "+ New Variable" → "Add Reference" → "DATABASE_URL"
PORT 3000 Port server (Railway biasanya auto-set)
Tips: Railway menyediakan variabel DATABASE_URL otomatis jika kamu klik "Add Reference" → pilih "DATABASE_URL" dari database PostgreSQL yang sudah ditambahkan. Tidak perlu copy-paste manual.

3.6 Deploy Backend

cd backend-kebab
railway up --service backend-kebab
Penjelasan perintah:
  • railway up — Upload dan deploy kode terbaru ke Railway
  • --service backend-kebab — Tentukan service mana yang di-deploy

Setelah deploy selesai, Railway akan menjalankan script:

npx prisma db push && node index.js

Ini akan membuat tabel-tabel di database secara otomatis, lalu menjalankan server.

3.7 Jalankan Migrasi Database (Jika Schema Berubah)

Setiap kali kamu mengubah file prisma/schema.prisma, jalankan:

cd backend-kebab

# Jalankan migrasi langsung di server Railway
railway run npx prisma db push --service backend-kebab
Peringatan: prisma db push akan mengubah struktur database secara langsung TANPA membuat file migrasi. Aman untuk development, tapi hati-hati di production karena bisa menghapus data jika ada perubahan yang tidak kompatibel.

3.8 Seed Data Awal — Buat Admin Login

Setelah database siap, kamu perlu membuat akun admin pertama secara manual.

Option A: Via Railway Shell

railway shell --service backend-kebab

# Di dalam shell Railway, jalankan Node.js
node -e "
const { PrismaClient } = require('@prisma/client');
const prisma = new PrismaClient();
async function main() {
  await prisma.admin.upsert({
    where: { username: 'admin' },
    update: {},
    create: { username: 'admin', password: 'password123' }
  });
  console.log('Admin berhasil dibuat!');
}
main().finally(() => prisma.\$disconnect());
"

# Ketik 'exit' untuk keluar dari shell
exit

Option B: Via API langsung (setelah backend jalan)

# Tambah data admin langsung ke database via Prisma Studio lokal
cd backend-kebab
npx prisma studio

Prisma Studio akan membuka di browser. Buka tabel Admin, klik "Add Record", isi username dan password, lalu klik "Save".

3.9 Cek Apakah Backend Sudah Jalan

# Cek log server
railway logs --service backend-kebab

# Atau buka URL backend di browser
# https://backend-kebab-production.up.railway.app

Jika backend sudah berjalan, kamu akan melihat response JSON dari endpoint root.

Bagian 4 — Deploy Landing Page ke Vercel

Landing page menggunakan Vue 3 + Vite + TailwindCSS.

4.1 Login ke Vercel

vercel login

Pilih metode login (GitHub, GitLab, Email, dll). Akan membuka browser.

4.2 Deploy Pertama Kali

cd frontend
vercel

Vercel akan bertanya:

  1. Set up and deploy? → Y (Yes)
  2. Which scope? → Pilih akun kamu
  3. Link to existing project? → N (No, buat baru)
  4. Project name? → frontend (atau nama lain)
  5. Directory? → Tekan Enter (default: ./)
  6. Override settings? → N
Penting: Vercel otomatis mendeteksi framework Vite dan mengisi build command (vite build) dan output directory (dist).

4.3 Set Environment Variable di Vercel

cd frontend
vercel env add VITE_API_BASE production

Masukkan URL backend Railway saat diminta:

https://backend-kebab-production.up.railway.app
Penjelasan: Variabel VITE_API_BASE akan dibaca oleh frontend/src/api.js saat build, sehingga frontend tahu alamat backend yang harus dihubungi.

4.4 Deploy ke Production

cd frontend
vercel deploy --prod --yes --force
Penjelasan flags:
  • --prod — Deploy ke domain production (bukan preview)
  • --yes — Skip konfirmasi (langsung deploy)
  • --force — Skip cache, build ulang dari awal

4.5 Cek Hasil

Setelah deploy selesai, Vercel akan menampilkan URL:

Production: https://frontend-phi-blond-37.vercel.app

Buka URL tersebut di browser untuk memastikan landing page tampil dengan benar.

Bagian 5 — Deploy Admin Dashboard ke Vercel

Admin Dashboard menggunakan Vue 3 + Vite + Custom CSS. Cara deploy mirip landing page.

5.1 Deploy Pertama Kali

cd admin-dashboard
vercel

Ikuti pertanyaan yang sama seperti landing page. Beri nama project: admin-dashboard

5.2 Set Environment Variable

cd admin-dashboard
vercel env add VITE_API_BASE production

Masukkan URL backend Railway:

https://backend-kebab-production.up.railway.app

5.3 Deploy ke Production

cd admin-dashboard
vercel deploy --prod --yes --force

5.4 Cek Hasil

Production: https://admin-dashboard-alpha-sage-15.vercel.app

Buka URL tersebut dan coba login dengan akun admin yang sudah dibuat di langkah 3.8.

Catatan: Admin dashboard dan landing page adalah 2 project terpisah di Vercel meskipun dalam 1 repository Git. Masing-masing punya domain sendiri.

Bagian 6 — Update Kode Setelah Perubahan

Setelah kamu mengubah kode dan ingin deploy ulang, ikuti langkah berikut:

Update Backend (Railway)

# 1. Masuk ke folder backend
cd backend-kebab

# 2. Deploy ulang
railway up --service backend-kebab
Tips: Jika kamu mengubah prisma/schema.prisma, Railway akan otomatis menjalankan prisma db push karena ada di script start. Jadi tabel akan ter-update otomatis.

Update Landing Page (Vercel)

# 1. Masuk ke folder frontend
cd frontend

# 2. Deploy ulang (paksa build fresh, skip cache)
vercel deploy --prod --yes --force

Update Admin Dashboard (Vercel)

# 1. Masuk ke folder admin-dashboard
cd admin-dashboard

# 2. Deploy ulang
vercel deploy --prod --yes --force
Peringatan Cache: Selalu gunakan --force saat deploy Vercel dari CLI. Tanpa flag ini, Vercel kadang memakai cache build lama sehingga perubahan kode tidak ter-deploy.

Workflow Lengkap Setelah Ubah Kode

# 1. Commit perubahan ke Git
git add .
git commit -m "deskripsi perubahan"
git push

# 2. Deploy backend (jika ada perubahan di backend-kebab/)
cd backend-kebab
railway up --service backend-kebab
cd ..

# 3. Deploy frontend (jika ada perubahan di frontend/)
cd frontend
vercel deploy --prod --yes --force
cd ..

# 4. Deploy admin (jika ada perubahan di admin-dashboard/)
cd admin-dashboard
vercel deploy --prod --yes --force
cd ..

Bagian 7 — Troubleshooting & FAQ

Masalah: Backend Error 502 / Server Crash

Penyebab umum:

  • Database belum dibuat / DATABASE_URL belum di-set
  • Prisma client belum di-generate
  • Port sudah dipakai oleh process lain

Solusi:

# Cek log error
railway logs --service backend-kebab

# Pastikan DATABASE_URL sudah ada di Railway variables
# Jika belum, tambahkan via dashboard Railway

Masalah: Frontend CORS Error

Penyebab: Backend belum mengizinkan domain frontend di CORS.

Solusi: Pastikan di backend-kebab/index.js, CORS sudah dikonfigurasi:

app.use(cors({
  origin: [
    'https://frontend-phi-blond-37.vercel.app',
    'https://admin-dashboard-alpha-sage-15.vercel.app',
    'http://localhost:5173'
  ]
}))

Masalah: Vercel Menampilkan Kode Lama

Penyebab: Vercel menggunakan cache build lama.

Solusi:

# Deploy ulang dengan --force untuk skip cache
vercel deploy --prod --yes --force

Masalah: API 404 di Railway (tapi jalan di lokal)

Penyebab: Kode terbaru belum ter-deploy ke Railway.

Solusi:

# Deploy ulang backend
cd backend-kebab
railway up --service backend-kebab

Masalah: Git Push Gagal (HTTPS)

Penyebab: Autentikasi HTTPS tidak bisa dilakukan di terminal.

Solusi: Push manual dari terminal terpisah atau gunakan GitHub Desktop:

# Buka terminal baru, lalu:
git push origin main

Masalah: Login Admin Gagal

Penyebab: Data admin belum ada di database.

Solusi: Buat admin pertama via Prisma Studio:

cd backend-kebab
npx prisma studio

Lalu buka tabel Admin → Add Record → isi username & password → Save.

Masalah: Database Kosong / Tabel Belum Ada

Solusi:

# Jalankan migrasi via Railway shell
railway shell --service backend-kebab
npx prisma db push
exit
